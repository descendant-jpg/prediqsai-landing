## PART C: DYNAMIC LINK LOADING

The app must load affiliate links from Supabase,
NOT from hardcoded values or environment variables.

On app startup:
1. Fetch active affiliate partners from Supabase:
   SELECT * FROM affiliate_partners 
   WHERE is_active = true
   ORDER BY book_name

2. Store in app state/context
3. Use these dynamic links everywhere in the app

This means when admin adds/edits/removes a partner
in the admin panel, it updates instantly in the app
without any code changes or app updates needed.

## SUPABASE TABLES TO CREATE

Run these SQL commands in Supabase:

CREATE TABLE affiliate_partners (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  book_name text NOT NULL,
  logo text,
  affiliate_url text NOT NULL,
  bonus_text text,
  commission_type text,
  commission_amount decimal,
  commission_currency text DEFAULT 'USD',
  min_payout decimal DEFAULT 0,
  payment_schedule text DEFAULT 'monthly',
  is_active boolean DEFAULT true,
  notes text,
  created_at timestamptz DEFAULT now(),
  updated_at timestamptz DEFAULT now()
);

CREATE TABLE affiliate_clicks (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id uuid REFERENCES auth.users(id),
  partner_id uuid REFERENCES affiliate_partners(id),
  book_name text,
  affiliate_url text,
  source text,
  clicked_at timestamptz DEFAULT now(),
  converted boolean DEFAULT false,
  commission_earned decimal DEFAULT 0,
  commission_currency text DEFAULT 'USD',
  notes text
);

CREATE TABLE affiliate_payouts (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  partner_id uuid REFERENCES affiliate_partners(id),
  amount decimal NOT NULL,
  currency text DEFAULT 'USD',
  paid_at timestamptz DEFAULT now(),
  payment_method text,
  reference text,
  notes text
);
