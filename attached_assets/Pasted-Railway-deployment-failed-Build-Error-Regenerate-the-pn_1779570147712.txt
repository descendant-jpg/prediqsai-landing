## Problem
The @workspace/website service fails during the BUILD stage with:
```
ERR_PNPM_LOCKFILE_CONFIG_MISMATCH  Cannot proceed with the frozen installation. The current "overrides" configuration doesn't match the value found in the lockfile
```
Railpack runs `pnpm install --frozen-lockfile --prefer-offline`, which requires the lockfile to exactly match the current workspace configuration. The SNAPSHOT stage succeeded; BUILD failed immediately during dependency installation.

## Diagnosis
The commit 36983e2 (by Replit Agent) added `@replit/connectors-sdk` to the root `package.json` and directly edited `pnpm-lock.yaml` to include the new dependency entries. However, directly editing the lockfile does not update the internal overrides metadata hash that pnpm uses to verify consistency. The `pnpm-workspace.yaml` defines extensive overrides for esbuild, lightningcss, rollup, tailwindcss/oxide, and expo/ngrok platform-specific packages. The lockfile's stored overrides configuration no longer matches what pnpm computes from `pnpm-workspace.yaml`, causing the `ERR_PNPM_LOCKFILE_CONFIG_MISMATCH` error.

## Fix
This fix cannot be auto-applied because it requires running `pnpm install` to regenerate the lockfile — a code agent can only edit text files, not execute commands.

1. **File:** `pnpm-lock.yaml` (repo root)
2. **Action:** Run `pnpm install` in the repo root to regenerate the lockfile
3. **Commands to run locally or in Replit shell:**
   ```bash
   pnpm install
   git add pnpm-lock.yaml
   git commit -m "Regenerate pnpm-lock.yaml to fix overrides mismatch"
   git push
   ```
4. **Why this fixes it:** Running `pnpm install` regenerates the lockfile from scratch, ensuring the overrides metadata hash matches the current `pnpm-workspace.yaml` configuration. The subsequent Railway build will then pass `pnpm install --frozen-lockfile` successfully.
