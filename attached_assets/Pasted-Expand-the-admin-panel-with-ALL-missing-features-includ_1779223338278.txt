UPDATE KEY FLOW:
When admin taps "Update Key":
1. Show input field
2. Admin pastes new key
3. App tests the new key first
4. If test passes → save to database
5. App uses new key immediately
6. Show success confirmation

Store API keys in app_config table:
INSERT INTO app_config 
(key, value) VALUES
('ANTHROPIC_API_KEY', 'sk-ant-xxx')
ON CONFLICT (key) 
DO UPDATE SET value = EXCLUDED.value;

Then read from database instead
of process.env for all API calls:
const getApiKey = async (keyName) => {
  const config = await db
    .select()
    .from(appConfig)
    .where(eq(appConfig.key, keyName))
    .limit(1);
  
  return config[0]?.value || 
    process.env[keyName];
};

TEST API BUTTON:
When admin taps "Test":
- Makes a test call to that API
- Shows response time
- Shows success or error
- Updates status badge instantly

TEST ALL APIS button:
- Tests all 10 APIs simultaneously
- Shows live results as they come in
- Highlights any failing APIs in red
- Sends email alert if any fail
